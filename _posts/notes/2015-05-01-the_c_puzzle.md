赋值操作法的规则从右到左
x *= y = z = 4     => x *= y = (z=4)
                          => x *= (y = (z=4))
                          => x = x * (y=(z=4))

逻辑操作符求值顺序从左到右
 x= x && y || z;          => x = (x && y ) || z

一元操作符的关系式从右到左 
z += - x ++ + ++ y     =>  z += - (x ++) + ( ++ y)   #tip1
                                   => z += (-(x ++)) + ( ++ y)
                                   => z = z +( (-(x ++)) + ( ++ y) )
#tip1 如果先绑定"-"操作符的话，表达式无法成立，++ -- 操作符要求以一个变量为操作数
x 是一个左值，但 -x 不是。

### 预处理器
预处理器并不是C语言的组成部分， 它的两个最重要的功能是对C程序里的宏命令进行替换和导入各种头文件
有操作符 括号起来
宏定义越紧凑越号，表达式比语句号，单条比多条好
避免副作用
要让对宏进行扩展的字符串成为一个完整的C语言元素
宏越简单越好，无法得到一个简单的宏，应该定义为函数
